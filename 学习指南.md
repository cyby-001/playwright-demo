## 📖 核心概念讲解

### 1. 什么是 Playwright？

**Playwright** 是微软开发的现代化浏览器自动化工具，相比 Selenium 有这些优势：
- 自动等待元素可见/可点击
- 执行速度更快
- API 更简洁
- 支持多种浏览器（Chrome, Firefox, Safari）

**核心概念：**
```python
from playwright.sync_api import Page

# Page 对象代表一个浏览器标签页
page.goto("https://example.com")  # 打开网页
page.locator("#button")           # 定位元素
page.click()                      # 点击
page.fill("text")                 # 输入文本
```

### 2. 什么是 Page Object 设计模式？

**问题：** 如果直接在测试代码中写页面操作，会出现：
- 代码重复（每个测试都要写一次登录）
- 难以维护（页面元素变了，要改很多地方）
- 可读性差（测试代码混杂页面细节）

**解决方案：Page Object 模式**

把每个页面封装成一个类：

```python
# pages/login_page.py
class LoginPage:
    def __init__(self, page: Page):
        self.page = page
        # 定义页面元素
        self.username_input = page.locator("#user-name")
        self.password_input = page.locator("#password")
        self.login_button = page.locator("#login-button")
    
    def login(self, username: str, password: str):
        # 封装登录操作
        self.username_input.fill(username)
        self.password_input.fill(password)
        self.login_button.click()
```

**好处：**
- 测试代码更简洁：`login.login("user", "pass")`
- 页面改动只需改一个地方
- 代码可读性更强

## 🔍 代码详细讲解

### 文件 1: `pages/login_page.py` - 登录页面对象

```python
from playwright.sync_api import Page

class LoginPage:
    # 类属性：页面的 URL
    URL = "https://www.saucedemo.com"
    
    def __init__(self, page: Page):
        """
        初始化方法
        :param page: Playwright 的 Page 对象（代表一个浏览器标签页）
        """
        self.page = page
        
        # 定义页面元素（使用 locator 定位）
        # locator 就像"遥控器"，告诉 Playwright 去哪里找元素
        self.username_input = page.locator("#user-name")    # 用户名输入框
        self.password_input = page.locator("#password")     # 密码输入框
        self.login_button   = page.locator("#login-button") # 登录按钮
        self.error_message  = page.locator("[data-test='error']")  # 错误消息
    
    def navigate(self):
        """打开登录页面"""
        self.page.goto(self.URL)
    
    def login(self, username: str, password: str):
        """
        执行登录操作
        :param username: 用户名
        :param password: 密码
        """
        self.username_input.fill(username)  # 输入用户名
        self.password_input.fill(password)  # 输入密码
        self.login_button.click()           # 点击登录按钮
    
    def get_error_message(self) -> str:
        """获取错误消息文本"""
        return self.error_message.inner_text()
```

**关键点：**
1. `__init__` 方法：初始化页面元素
2. `page.locator()`：Playwright 的元素定位方法，支持 CSS 选择器、XPath 等
3. `fill()` / `click()`：Playwright 的操作方法

### 文件 2: `tests/test_login.py` - 登录测试用例

```python
import pytest
from pages.login_page import LoginPage

# 测试用例 1: 登录成功
def test_login_success(page):
    """
    测试标准用户登录成功
    :param page: pytest fixture 提供的 Page 对象（定义在 conftest.py）
    """
    # 创建 LoginPage 对象
    login = LoginPage(page)
    
    # 执行操作
    login.navigate()                      # 打开登录页
    login.login("standard_user", "secret_sauce")  # 登录
    
    # 断言验证
    assert "/inventory" in page.url, "登录成功后应跳转到商品页"

# 测试用例 2: 登录失败（参数化测试）
@pytest.mark.parametrize("username, password, expected_error", [
    ("standard_user",  "wrong_pass",   "Epic sadface"),           # 密码错误
    ("",               "secret_sauce", "Username is required"),   # 用户名为空
    ("standard_user",  "",             "Password is required"),   # 密码为空
    ("locked_out_user","secret_sauce", "Sorry, this user has been locked out"),  # 被锁定用户
])
def test_login_fail(page, username, password, expected_error):
    """
    参数化测试：用不同的数据测试登录失败场景
    pytest.mark.parametrize 会自动生成多个测试用例
    """
    login = LoginPage(page)
    login.navigate()
    login.login(username, password)
    
    error_text = login.get_error_message()
    assert expected_error in error_text, f"期望报错包含: {expected_error}，实际: {error_text}"
```

**关键点：**
1. **pytest fixture**：`page` 参数由 `conftest.py` 提供
2. **参数化测试**：`@pytest.mark.parametrize` 用一组数据运行同一个测试
3. **断言**：`assert` 语句验证测试结果

### 文件 3: `conftest.py` - pytest 配置

```python
import pytest
from playwright.sync_api import Page

@pytest.fixture
def page(browser):
    """
    pytest fixture: 提供 Page 对象
    这个 fixture 会被所有测试函数自动使用
    """
    page = browser.new_page()  # 创建新页面
    yield page                 # 提供给测试使用
    page.close()               # 测试结束后关闭页面
```

**什么是 fixture？**
- pytest 的 fixture 是"测试前置/后置"的优雅实现
- 比如：测试前登录、测试后清理数据
- `page` fixture 让你可以直接在测试函数中用 `page` 参数

## 💡 常见问题解答

### Q1: 我不知道从哪里开始学？
**A:** 按这个顺序：
1. 先运行一次测试，看到效果：`pytest tests/test_login.py -v`
2. 打开浏览器开发者工具（F12），对照代码看元素定位
3. 修改一个小地方（比如测试数据），再运行看效果

### Q2: `page.locator()` 里的 `#user-name` 是什么？
**A:** 这是 CSS 选择器：
- `#` 表示 id
- `.` 表示 class
- 比如 `#user-name` 就是找 `id="user-name"` 的元素

你可以在浏览器按 F12，用元素选择器（左上角箭头）点击页面元素查看。

### Q3: 为什么测试函数参数是 `page`？
**A:** 这是 pytest 的 fixture 机制：
- `conftest.py` 定义了 `page` fixture
- pytest 自动把 fixture 的返回值传给测试函数
- 所以你直接用 `page` 就行，不用自己创建

### Q4: 我想加新功能，怎么下手？
**A:** 按这个步骤：
1. 在浏览器手动操作一遍，理解流程
2. 用 F12 查看相关元素的定位方式
3. 在 `pages/` 创建新的 Page Object 类
4. 在 `tests/` 写测试用例
5. 运行测试调试
